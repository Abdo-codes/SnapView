# Security Policy

If you believe you have found a security issue in `snapview`, do not open a public GitHub issue with exploit details.

Instead:

- report the issue privately to the project maintainers
- include reproduction steps, impact, and any proof-of-concept details needed to validate the report
- allow time for triage and a coordinated fix before public disclosure

Security issues are handled separately from normal bugs because this project interacts with local developer tooling, build artifacts, and test hosts.

For non-sensitive bugs or product issues, use the normal issue tracker instead.
